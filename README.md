# Go Payment Core

Мой проект для практики бэкенд-разработки на Go. Написал базовую логику для обработки платежей в одном файле.

### Что уже сделано:
* **Транзакции:** Создал структуру платежа. Данные хранятся в памяти через мапу (`map`), для каждой операции создается свой ID.
* **Функции:** Написал метод `charge` для проведения оплаты с проверкой суммы и метод `refund` для отмены платежа по его ID.

### 🚀 Планы на будущее:

* **Сделать параллельную обработку (Многопоточность):**
  - Буду разбираться с горутинами (`goroutines`) и каналами (`channels`), чтобы платежи обрабатывались пачками в несколько потоков, а не по очереди.
  - Подключу `Mutex` и `WaitGroup` из пакета `sync`, чтобы защитить данные от сбоев и ошибок («состояния гонки»), когда одновременно списываются деньги.
  - Буду проверять код через детектор гонок `go test -race` и смотреть нагрузку через `pprof`.

* **Вывести проект в сеть (Сделать API):**
  - Напишу полноценный веб-сервер на стандартном пакете `net/http` (или роутере `chi` / `gin`), чтобы принимать реальные запросы.
  - Сделаю нормальный прием и обработку данных в формате JSON.
  - Разберусь с контекстами (`context.Context`), чтобы вовремя отменять зависшие сетевые запросы и платежи.
  - На будущее хочу попробовать gRPC для быстрой связи между сервисами.

* **Подключить базу данных и навести порядок в коде:**
  - Перенесу хранение транзакций из оперативной памяти в настоящую БД PostgreSQL.
  - Настрою SQL-транзакции, чтобы балансы никогда не ломались при сбоях.
  - Прикручу Redis для быстрого кэширования частых запросов.
  - Перепишу код по слоям (Чистая архитектура) и раскидаю файлы по правильным папкам: `cmd/`, `internal/`, `pkg/`.

* **Инфраструктура и мониторинг:**
  - Упакую весь проект, базу данных и кэш в контейнеры через Docker и Docker Compose, чтобы всё запускалось одной командой.
  - Сделаю человеческие логи в формате JSON через пакеты `zap` или `zerolog`, чтобы видеть ошибки.
  - Хочу прикрутить Prometheus и Grafana, чтобы выводить графики успешных платежей в реальном времени.
  - Настрою GitHub Actions, чтобы тесты запускались сами при каждом пуше.
